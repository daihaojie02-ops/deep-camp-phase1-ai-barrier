## Prompt 设计原则

1. **先整体后细节：** 先说风格和结构，再列具体数据
2. **"不需要"比"需要"更重要：** 明确指出不要什么（交互 JS/复杂动画/外部框架）——避免 Claude Code 过度生成
3. **数据先行：** 如果第一次 Prompt 就带真实数据，可以省掉第二轮的修改
4. **验收标准明确：** "打开浏览器看到..."——可验证的验收条件
