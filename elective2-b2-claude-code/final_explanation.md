## 4. 哪些地方是我自己判断、修改或验收的？

1. **设计方向决策：** 我决定用单文件 HTML 而非多文件项目——因为这只是展示页面，不需要复杂结构
2. **数据准确性：** 所有任务名称、评分、截止日期都是我自己核对后填入的——Claude Code 不知道这些真实数据
3. **风格要求：** "暗黑极客风格"、"简洁不花哨"——这些是我的审美判断
4. **截止日期颜色：** 我要求将快到期任务标红——这是对"信息层级"的判断
5. **验收：** 浏览器打开后，逐条对比我的实际任务状态确认无误

1. **动态数据：** 用 JS 读取一个 JSON 配置文件，修改数据不用改 HTML
2. **交互功能：** 点击任务展开详情、勾选完成状态
3. **GitHub Pages 部署：** 让页面可以通过 URL 访问，而非本地打开文件

**但当前阶段不需要。** 这个页面是给我自己看的，7 个任务固定，手动更新足够了。过度优化一个展示页是浪费精力。
