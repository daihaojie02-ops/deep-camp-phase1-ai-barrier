# B2 Claude Code 使用反思

> 戴昊杰 (DEEP007) | 2026.07.15

## 1. Claude Code 最适合帮我解决什么问题？

**文件级操作——生成、修改、组织项目文件。** 不是"聊天给建议"，而是直接产出可运行/可查看的文件。

本次 B2 任务中，Claude Code 在 30 秒内生成了完整的 HTML/CSS 网页文件。如果让我手动写，我完全不会写 CSS——这个任务本来对我是不可能的。

**最适合的场景特征：** 需求明确、输出目标清晰、可以用自然语言描述完整。最不适合的场景：需求模糊、需要多轮探索才能定义问题。

## 2. 它和普通 ChatGPT 对话有什么不同？

| 维度 | ChatGPT 对话 | Claude Code |
|------|------------|-------------|
| 输出形式 | 文字回复 | 项目文件（.html/.py/.js 等） |
| 后续操作 | 复制粘贴 → 手动保存 | 文件直接在项目中可用 |
| 上下文 | 每次对话独立 | 可以读取/修改已有文件 |
| 验证方式 | 凭阅读判断 | 浏览器打开/终端运行 |
| 适用场景 | 咨询、解释、头脑风暴 | 执行、生成、修改、部署 |

**一句话：ChatGPT 对话给你想法，Claude Code 给你结果。** 本次 B2 任务如果用 ChatGPT，得到的是一段"好的，这是你的 HTML 代码..."——还需要我手动复制保存。Claude Code 直接写了文件到 `project/index.html`。

## 3. 我在使用中遇到的最大困难是什么？

**不是 Claude Code 的问题——是我的问题：如何描述一个"好"的需求。**

第一次 Prompt 我描述了功能和风格，但忘了给真实数据——导致 Claude Code 用示例数据填充。这不是 Claude Code 的错——它不知道我的真实任务信息。

**核心困难：需求描述的完整度。** 给 Claude Code 的信息越完整，第一版就越接近目标。遗漏信息 = 需要多轮迭代。

## 4. 我是否理解了它生成的所有内容？

**CSS 部分——不完全理解。** 我理解 HTML 结构（table/tr/td/div），但 CSS 中的 `linear-gradient`、`border-collapse`、`transition` 等属性是 Claude Code 生成的，我大致知道作用但不完全理解语法。

**但这不影响使用。** 我不是前端开发者——我不需要会写 CSS。我需要的是：知道 CSS 控制网页外观、能向 Claude Code 描述"把背景改成暗色"的需求、能验证修改是否生效。

## 5. 如果别人也能使用 Claude Code，我和他们的差距在哪里？

**差距不在于"会不会用 Claude Code"，而在于"能用它做出什么"。**

别人用 Claude Code：生成一段代码 → 复制 → 不知道能不能跑 → 放弃。
我用 Claude Code：
1. 先想清楚要解决什么问题（不是先打开 Claude Code）
2. 把问题描述成可执行的需求（清晰、具体、有验收标准）
3. 让 Claude Code 生成文件
4. 在浏览器/终端验证结果
5. 发现问题 → 描述问题 → 让 Claude Code 修改
6. 最终产出整理到 GitHub

**差距是"从需求到产出的完整闭环"，不是"谁会写更好的 Prompt"。**

## 6. 以后我准备如何继续提高 Claude Code 使用能力？

1. **每次使用后记录"Prompt 模板"：** Task 003 的 Prompt 模板方法论可以直接用在这里——把每次成功的使用提炼为可复用模板
2. **扩大使用场景：** 不只是生成网页——生成 Python 脚本（B3）、整理仓库结构、生成 README
3. **学习命令行：** Claude Code 是在终端里跑的——对终端的理解越深，使用效率越高
4. **关注 Anthropic 更新：** Claude Code 功能更新快，不追踪就会错过新能力
