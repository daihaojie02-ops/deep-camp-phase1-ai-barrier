# B1 信息主题选择：AI Agent 开发工具与 MCP 生态

> 戴昊杰 (DEEP007) | 2026.07.15

## 1. 我选择的主题是什么？

**AI Agent 开发工具与 MCP（Model Context Protocol）生态发展。**

具体追踪对象：
- Claude Code / Codex / Cursor 等 AI 编程工具的能力演进
- MCP Server 生态——哪些新 Server 出现、哪些被社区验证、哪些被淘汰
- AI Agent 框架（LangChain/CrewAI/Microsoft Agent Framework/OpenAI Agents SDK）的竞争格局
- AI 编程工具对"非代码背景学习者"的实际门槛变化

## 2. 为什么这个主题值得关注？

三个原因：

**第一，Agent 是 2026 年 AI 最主流的方向。** MCP 已被 Anthropic 发布、被 OpenAI 和 Google 采纳、被捐赠给 Linux Foundation，月 SDK 下载量达 1.1 亿次。这不是一个框架或一个公司的策略，是整个 AI 行业在统一"AI 如何与外部世界交互"的标准。

**第二，编程工具是能力放大器。** 不会代码的人用 ChatGPT 能生成文字，但用了 Claude Code 能生成可运行的项目。这个差距不是"代码能力"的差距，是"工具选择"的差距。我需要持续追踪哪些工具降低了非程序员的使用门槛。

**第三，信息差决定了选型质量。** Task 002 我选了 CrewAI 入门 Multi-Agent——这个决策做出来是因为我当时搜到了 LangChain 官方对比文章。如果我没有主动搜，可能选了更不适合我的框架。信息差直接转化为决策质量。

## 3. 这个主题和我的 AI 壁垒方向有什么关系？

Task 001 我确定的壁垒方向是 **"Multi-Agent 工程能力"**。

这个方向的根基不是某一个框架的使用技巧，而是：
- 知道有哪些 Agent 工具可用（信息差）
- 能判断哪个工具适合我的场景（判断力）
- 能追踪工具生态变化、及时调整策略（信息雷达）

本主题直接支撑这个壁垒方向的第一层和第二层：信息差 + 判断力。

## 4. 这个主题可能对哪些方面有帮助？

| 方向 | 具体帮助 |
|------|---------|
| 学习 | 避免盲目追新工具——先了解生态再决定投入时间 |
| 比赛 | 很多比赛要求 Agent 实现，工具选对了效率翻倍 |
| 项目 | MCP Server 可以复用——一个 Server 配置多个项目共用 |
| 创业 | Agent 开发和 MCP 集成是独立招聘岗位，有市场需求 |
