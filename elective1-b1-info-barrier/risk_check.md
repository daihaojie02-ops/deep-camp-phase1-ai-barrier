# B1 成本、门槛与风险分析

> 戴昊杰 (DEEP007) | 2026.07.15

## 一、工具使用成本

| 工具 | 费用 | 隐藏成本 |
|------|------|---------|
| Claude Code | 需 Anthropic API 额度（或 Claude 订阅） | Token 消耗可控但需关注——Task 002 一次调研约消耗 $2-5 |
| Firecrawl MCP | 免费额度（搜索+抓取） | 重度使用需付费，Task 003 测试运行消耗了约 5 credits |
| Playwright MCP | 免费 | 需要安装浏览器依赖 |
| GitHub | 免费 | 学习 git/gh CLI 命令的时间成本 |

**我的判断：** 目前这套工具栈月成本 <$20（Anthropic API + Firecrawl 免费额度内），对一个学习者来说完全可接受。但如果未来做深度调研频率增加，需要评估 Anthropic API 的月度预算。

|--------|---------|---------|

## 三、学习门槛

| 技能 | 难度 | 我的状态 |
|------|------|---------|
| 命令行基础（cd/ls/mkdir） | 低，1天入门 | ✅ 已掌握 |
| MCP 配置文件编写（JSON） | 低，参考模板即可 | ✅ 已配置 |
| 理解 Agent 框架概念（Agent/Tool/Task/Orchestration） | 中，需阅读文档+实践 | ⚠️ Task 002 已入门，但 CrewAI 的异步执行等高级特性还不熟 |
| 判断 Claude Code 生成代码的质量 | 高，需要一定代码理解 | ⚠️ 能判断"能不能跑"，但判断"代码好不好"还需要学习 |

**我的判断：** 前 3 项已经越过。第 4 项是长期功课——不是几周能解决的，但不影响当前阶段的使用。

## 四、合规与隐私风险

| 风险类型 | 具体风险 | 严重程度 |
|---------|---------|---------|
| API 中转站使用 | 有些同学用低价 API 中转——可能涉及账号共享、数据泄露 | ⚠️ 中——我不使用中转站 |
| MCP Server 权限 | MCP Server 可能读取本地文件系统——需要审计 Server 代码 | ⚠️ 中——我只用知名开源 Server（Firecrawl/Playwright） |
| AI 生成内容版权 | Claude Code 生成的代码版权归属——目前法律灰色地带 | 🔵 低——个人学习用途风险低 |
| 隐私数据上云 | 用 Firecrawl 抓取的内容和 Claude API 调用的数据流经第三方 | 🔵 低——不涉及个人敏感数据 |

**我的立场：** DEEP 营鼓励合法合规可持续的信息差。我不碰 API 中转、不碰灰色工具、不用盗版。

## 五、过度宣传/夸大效果？

| 工具/概念 | 宣传话术 | 真实情况 |
|----------|---------|---------|
| "17,000+ MCP Server" | "生态超繁荣" | 绝大多数是 PoC/已放弃项目——信号噪声比极差（Task 003 实测） |
| "AI Agent 自动完成复杂任务" | "替代人工" | Agent 仍然需要人工监督、验证和纠错——不是"设好就忘" |
| "零代码 AI 开发" | "完全不需要会代码" | 完全零代码很难做可靠工具——至少需要理解命令行和文件结构 |

## 六、是否只是短期热点？

| 方向 | 短期热度判断 | 长期价值判断 |
|------|------------|------------|
| Agent 框架 | 2026 年最热赛道——但框架会继续洗牌 | 协议标准（MCP/A2A）比具体框架更重要——学协议不学单一框架 |
| AI 编程工具 | Claude Code 热度上升中 | 编程工具是"能力放大器"——不是热点，是新常态 |
| MCP Server | 数量爆发但质量参差 | 标准本身在稳固（Linux Foundation 托管）——长期有价值 |

## 七、普通学员是否适合现在投入时间？
