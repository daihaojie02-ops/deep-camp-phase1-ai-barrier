# 真实场景说明

## 场景类型
**信息检索场景** — 深度技术调研报告

## 场景具体描述

我需要在 DEEP 营第一期 Task 3（AI 工作流与 Agent 雏形搭建）开始前，选定一个 Multi-Agent 框架来深度学习。

当前有 4 个主流选择：CrewAI（54.5K⭐）、MetaGPT（69.1K⭐）、AutoGen（59.3K⭐）和 Microsoft Agent Framework（11.7K⭐，2026.04 GA）。仅凭 GitHub 星数排序，MetaGPT 最高；但 Task 1 调研时我已经发现 AutoGen 进入了维护模式——说明"星数 ≠ 该不该学"。

不靠系统调研，我只凭直觉或一篇 2025 年的旧文章盲选。选错了框架，可能浪费一周时间学一个不适合我场景（或者已停止维护）的工具。因为：
1. 不同的框架有不同的设计哲学（角色驱动 vs 对话驱动 vs 图编排）
2. 不同框架适合的场景差异巨大（内容创作 vs 软件工程 vs 企业级）
3. 我的基础薄弱（Multi-Agent 自评 1/5），选一个学习曲线太陡的框架可能半途而废

**不靠 AI 时我原本会怎么做：**
打开 Google → 搜 "CrewAI vs MetaGPT" → 看第一篇 Medium 文章 → 凭文章结论做决定。整个过程大概 30 分钟，但判断质量完全取决于我碰到的第一篇文章的质量。

**不靠 AI 时困难在哪里：**
1. 搜索引擎给出的对比文章往往是 2024-2025 年的，可能不包含 MAF GA 这个关键信息
2. 我没有 GitHub API 直接对比 star/issue/fork 的增长趋势
3. 看一篇对比文章等于接受一个人的观点，没有交叉验证

## 这个问题真实存在在哪

**是我自己的问题：** DEEP 营 4 个任务中，Task 3 直接依赖 Multi-Agent 框架选型。我的自评矩阵中 Multi-Agent 编排是 1/5——我是真正的新手。

**别人也可能遇到：** 任何在 2026 年入场学 Multi-Agent 的人，面对的选择都是相同的 4-5 个框架（加上 LangGraph）。信息过载 + 框架快速迭代 + 大量过时内容 = 决策困难。

## 为什么这个场景适合 AI 应用实验

1. **信息检索是 AI 的强项。** Firecrawl 可以直接抓取 GitHub 页面和技术博客的最新数据，时效性远超搜索引擎
2. **需要结构化输出。** 不是"搜到一堆文章"，而是"按统一框架对比分析"——这正是 Claude 擅长的
3. **有明确的"不用 AI vs 用 AI"对比空间。** 30 分钟盲选 vs 2 小时系统调研
4. **产出可公开验证。** 报告的结论可以被其他人检查和挑战
