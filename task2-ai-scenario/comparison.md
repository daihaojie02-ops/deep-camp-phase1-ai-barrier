# AI 前后效果对比

## 对比总览

| 对比项 | 不用 AI | 使用 AI 后 | 我的判断 |
|--------|---------|-----------|---------|
| **时间成本** | 约 30 分钟（Google 搜→看一篇对比文章→拍脑袋决定） | 约 2.5 小时（系统检索→数据抓取→报告撰写→修改→复盘） | 看起来 AI 花了更多时间，但 30 分钟的版本只是一次性决策，AI 版本产出了可复用的报告 + 可追溯的判断依据。**时间投入和质量产出不是线性关系。** |
| **信息广度** | 1-2 篇对比文章，取决于搜索结果排名 | 4 个框架的 GitHub API 实时数据 + 多篇对比文章 + 社区讨论 + 学术论文引用 | AI 版本多了一个信息维度（API 数据），而且时效性（2026.06.29）远超 2024 年的对比文章 |
| **信息深度** | 接受文章作者的结论，没有交叉验证 | 每个框架独立分析 + 横向对比矩阵 + 个人判断段落 | 不用 AI 时我是"信息的消费者"；用 AI 后我是"信息的整合者和判断者" |
| **结构清晰度** | 脑内零散记忆，没有结构化输出 | 8 章节 Markdown 报告 + 对比决策矩阵 + 学习路线图 | 结构化的最大价值不是"好看"，而是"别人能检查我的判断是否有漏洞" |
| **错误风险** | 高：如果搜到的第一篇文章是 2024 年的，不会知道 AutoGen 已停止维护 | 中：API 数据是实时的，错误主要来自我的个人判断偏差 | AI 降低了信息过时风险，但没有消除判断偏差——我仍然可能对某个框架有未经验证的偏见 |
| **可复用性** | 零——30 分钟后除了脑中的印象什么都不剩 | 高——报告可以直接作为 Task 3 的前置文档，也可以分享给其他 DEEP 学员 | **这是 AI 辅助调研的核心价值：一次投入，持续产出** |

## AI 哪一步最有帮助

**GitHub API 数据批量抓取 + 结构化对比。** 手动打开 4 个 GitHub 页面、记录数据、画对比表——这个过程至少 30 分钟且容易出错。`gh api` 一次调用 2 秒，数据准确、格式统一。这是 AI 工具最擅长的事：把重复机械操作自动化。

## AI 哪一步不可靠

**"我的判断"段落。** AI 可以生成看起来像个人判断的文本（比如"A 框架适合初学者"），但它不知道我的技术背景、我的学习习惯、我的 Task 3 计划。真正的个人判断必须我自己写——AI 只是给我提供了一个写判断的框架。

## 哪些地方必须由我自己判断

1. **最终推荐。** AI 分析了 4 个框架的优劣，但"选哪个"是我的决策，需要结合我自身的情况（基础 1/5、Task 3 计划、学习时间）
2. **信息盲区。** AI 不会主动承认"我用的训练数据可能不包含最新信息"——识别盲区是人的责任
3. **场景匹配。** AI 不知道我的 Task 3 具体方向，无法判断哪个框架和我的场景最匹配

## 个人修改痕迹

### 修改 1：从"百科式介绍"到"有我判断的分析"

**AI 原始输出的问题：** Claude 生成的框架分析初稿偏向百科风格——"CrewAI 是一个基于角色的多 Agent 框架，支持 Sequential、Hierarchical 等模式..."——读起来像官方文档，看不出作者是谁。

**我做的修改：** 在每个框架分析末尾增加了"我的判断"段落。以 CrewAI 为例，我写了：
> "我对刚入门 Multi-Agent 的人来说，它是最安全的起点——不是因为星数多，而是因为它的心智模型最简单..."

**为什么这样修改：** Task 1 的 Auth 评审明确指出"没有你的判断，仓库再整齐也只是空壳"。一份技术调研报告的核心价值不在于罗列数据（AI 可以做），而在于结合自身情况做选择（只有人可以做）。

**修改后的效果：** 报告从"介绍各框架"升级为"帮读者（和我自己）做出选择"。评审者能看到我是一个有判断力的人，而不是一个会搜信息的 AI 操作员。

### 修改 2：增加"信息盲区"章节

**AI 原始输出的问题：** Claude 的报告初稿没有主动列出不确定的内容。它在生成对比表格时表现得"什么都懂"，但实际上有些判断是基于训练数据的推测而非验证过的信息。

**我做的修改：** 在报告末尾新增第八章"信息盲区（我承认我还无法判断的）"，列出了 4 条我没有验证或不确信的内容。例如：
> "CrewAI Flows 的实际稳定性——我只看了文档没实际跑过，不确定在生产环境中是否稳定。"

**为什么这样修改：** 承认盲区不是示弱——在 AI 时代，知道自己不知道什么比假装知道更稀缺。这也是对评审者的诚实：不宣称自己做了超出实际范围的验证。

**修改后的效果：** 报告的可信度反而提高了。一份承认盲区的报告比一份"全知全能"的报告更可靠。

### 修改 3：重新设计对比矩阵的维度

**AI 原始输出的问题：** Claude 初稿的对比表格维度偏向技术指标（性能、架构、语言兼容性），缺少"适合我的程度"这种个人化维度。

**我做的修改：** 在横向对比决策矩阵中增加了"学习曲线"、"场景灵活性"、"长期维护保障"三个维度，最后一行加了"**适合我的程度**"——这个维度是 AI 不可能填的，只有我自己能填。

**为什么这样修改：** 技术调研如果只有"客观指标"就只是信息整理。加上"适合我的程度"这个主观维度，调研才变成了决策支持。

**修改后的效果：** 对比矩阵从技术对比表变成了决策工具。其他 DEEP 学员看同一张表可以得出不同的结论（取决于他们自己的背景）——这正是"有个人判断"和"复制 AI 输出"的区别。

### 修改 4：删除无效内容，压缩冗余段落

**AI 原始输出的问题：** Claude 初稿中部分段落有 Gen-AI 常见的"总结性废话"——比如在 CrewAI 分析中有一段"总之，CrewAI 通过角色驱动模型降低了 Multi-Agent 开发的门槛..."，这实际上只是把前文重复了一遍。

**我做的修改：** 删除了所有"总之""综上所述"开头的总结性段落。如果前面已经说清楚了，不必再总结。

**为什么这样修改：** AI 倾向于在每段末尾加一个"过度总结"——这是训练数据中大量存在的模式。但对于一个已经有清晰结构的报告来说，冗余总结只会稀释信息密度。

**修改后的效果：** 报告更紧凑，每个章节有实质增量信息。

## 这个方法下次是否可复用

**完全可以。** 这次的"信息检索 → 结构化对比 → 个人判断"流程是一个通用模板：
- 下次需要对比任何技术选型（数据库、前端框架、部署方案），直接套用同一流程
- 关键变量只有两个：对比维度（需要根据领域调整）和信息源（需要根据时效性更新）
- Task 3 的复盘部分可以直接引用本报告的方法论
