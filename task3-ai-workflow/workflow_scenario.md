# 工作流场景说明

> 戴昊杰 (DEEP007) | 2026.07.05 | Task 003

---

## 1. 这个工作流解决什么问题？

**解决"面对陌生技术领域，如何从一无所知到做出有理由的决策"这个问题。**

具体来说：当你遇到一个新技术领域（比如一个新的开源框架、一个新的工具品类、一个新的技术趋势），你需要在有限时间内搞清楚：
- 这个领域有哪些主要玩家？
- 它们之间的关键差异是什么？
- 社区健康度如何（是否值得投入时间学习）？
- 基于你的个人约束条件，哪个选择最合理？
- 有哪些东西你现在还无法判断（盲区）？

传统做法是 Google 搜索 → 看几篇文章 → 凭感觉选一个。这个工作流把上述模糊过程变成了 6 步标准作业程序，每一步有明确的输入、输出和检查标准。

---

## 2. 这个问题是否会反复出现？

**会，而且是高频出现。**

- 作为开发者，每个季度都有新的框架、工具、方法论涌现
- 作为学习者，每进入一个新领域都需要快速建立认知框架
- Task 002 是 Multi-Agent 框架选型，未来可能是 RAG 框架选型、Agentic 平台选型、MCP Server 工具链选型、AI 评测框架选型……
- 问题结构完全一样：多候选→需要对比→有个人约束→需要做决定

Task 002 的实际数据：无 AI 时预估需要 215 分钟，有 AI 辅助但无结构化流程花了 97 分钟。如果有了这套工作流，下一次类似的调研预计可以压缩到 60 分钟以内。

---

## 3. 谁可能使用这个工作流？

**第一用户是我自己**——每次面对新技术选型时直接复用。

**潜在第二用户是类似的开发者/学习者**——任何需要在陌生技术领域快速建立认知并做出选择的人：
- 自学编程的新手（不知道该学哪个框架）
- 做技术选型的 junior 开发者
- 需要快速了解一个技术赛道的产品经理
- DEEP 营后续学员（003 的工作流可以直接用在 004、005 的信息检索环节）

---

## 4. 每次使用前需要准备什么材料？

| 输入项 | 说明 | 示例（来自 Task 002） |
|--------|------|----------------------|
| 调研主题 | 一句话描述要调研什么 | "2026 H1 Multi-Agent 框架生态" |
| 候选列表 | 已知的主要玩家（可以为空，工作流会补充） | CrewAI / MetaGPT / MS Agent Framework / AutoGen |
| 个人约束条件 | 你的技术水平、预算、时间、特殊需求 | 新手、非.NET、Token敏感、偏好Python生态 |
| 调研目标 | 最终要做一个什么决定 | "选一个框架开始学习" |
| 已知信息差 | 你目前知道什么、不清楚什么 | "知道星数、不知道差异" |

如果候选列表为空也没关系——工作流第一步就是信息检索，会自动补充候选列表。

---

## 5. 使用这个工作流后，最终应该得到什么结果？

一份**结构化调研报告**，包含：

1. **调研动机与范围说明** — 我为什么要做这个调研、圈定了哪些候选
2. **生态概览** — 领域现状、关键趋势
3. **逐候选深度分析** — 每个候选的定位/核心特点/社区健康度/适用场景/局限
4. **横向对比矩阵** — 10+维度统一尺度的对比表
5. **个人判断与推荐** — 基于我的约束条件做出的选择 + 选择理由
6. **学习路线图** — 选了什么 + 怎么开始 + 第一步做什么
7. **信息盲区标注** — 哪些信息我无法验证 + 为什么 + 如果继续需要做什么
8. **来源列表** — 全部信息来源，可追溯核查

最终输出格式是标准化的 Markdown 文档，可以直接放入 GitHub 仓库作为资产积累。

---

## 6. 这个工作流和你的个人 AI 壁垒有什么关系？

**这是壁垒从"概念"到"方法"的关键一步。**

```
Task 001 说：我要做 Multi-Agent 工程能力 → 这是方向（概念）
Task 002 做：一次具体的框架调研 → 这是证明（我做到过一次）
Task 003 建：可复用的调研工作流 → 这是方法（我能反复做到）
```

没有工作流，Task 002 的调研经验会随时间衰减——三个月后换一个领域，又要重新摸索。有了工作流，每次调研都在同一套框架下进行，积累的不是散落的知识点，而是不断升级的判断力。

**这个工作流本身就是壁垒的"操作系统"**——它不是单一技能，而是生产技能的方法。普通 AI 使用者拿到 ChatGPT 问一句得到一个答案，我拿着这套工作流问一轮得到一个决策依据。输出的层次完全不同。
