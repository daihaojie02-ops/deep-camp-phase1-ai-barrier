# Agent 雏形说明

> 戴昊杰 (DEEP007) | 2026.07.05 | Task 003

---

## 我的 Agent 雏形定义

**名称：** "TechScout" — AI 技术调研 Agent（普通版）

**类型：** Prompt + SOP + 模板驱动（非代码自动化 Agent）

---

## 1. 我的 Agent 雏形解决什么任务？

**解决"面对陌生技术领域时，快速从信息检索到有理由决策"的完整链路。**

具体任务：用户提供一个技术调研主题 + 个人约束条件 → Agent 按标准流程完成信息检索、分析、对比、质量检查和结构化输出 → 产出一份可追溯、可复用、有明确个人判断的调研报告。

这不是一个"帮我搜一下 XX"的简单问答，而是一套端到端的调研决策系统。

---

## 2. 它有哪些固定输入？

| 输入项 | 类型 | 示例 |
|--------|------|------|
| 调研主题 | 文本（必填） | "2026年Python Web框架对比" |
| 候选列表 | 文本列表（选填，可为"待发现"） | ["FastAPI", "Django", "Flask", "Litestar"] |
| 个人约束条件 | 结构化文本（必填） | 技术水平 / 预算 / 语言偏好 / 时间 |
| 调研目标 | 一句话（必填） | "选一个框架开始学习REST API开发" |
| 已知信息差 | 文本（选填） | "知道名字，不知道异步性能差异" |

---

## 3. 它按什么步骤执行？

```
Step 1: 任务拆解
  → 使用 Prompt 模板 1（任务拆解）
  → 产出调研方案：维度列表 + 信息源规划

Step 2: 信息检索
  → Firecrawl 多源搜索 + GitHub API 数据抓取
  → 产出原始信息集合

Step 3: AI 结构化提取
  → 使用 Prompt 模板 2（内容生成/处理）
  → 产出结构化分析草稿

Step 4: 人工判断注入 ⭐
  → 人类操作者注入个人约束 + 盲区标注
  → 产出含判断的分析稿

Step 5: 横向对比 + 报告生成
  → 使用 Prompt 模板 2（再次调用，生成对比矩阵和完整报告）
  → 产出报告初稿

Step 6: 质量检查
  → 使用 Prompt 模板 3（质量检查）+ 人工逐项核对
  → 产出问题列表 + 修改后的终稿

Step 7: 复盘迭代
  → 记录本轮发现的问题 → 更新 Prompt 模板和检查表
```

每一步有明确的输入、处理方式、输出和检查标准，详见 `workflow_diagram.md`。

---

## 4. 它最终输出什么？

一份标准化的 8 章技术调研报告（详见 `output_template.md`）：

1. 调研动机与范围
2. 生态概览
3-N. 候选深度分析
N+1. 横向对比矩阵
N+2. 个人判断与推荐
N+3. 学习路线图
N+4. 信息盲区与限制
来源列表

输出格式：Markdown 文档，可直接放入 GitHub 仓库。

## 5. 哪些环节仍然需要人工判断？

| 环节 | 为什么必须人工 | 交给AI的风险 |
|------|---------------|-------------|
| **Step 4: 人工判断注入** | 个人约束条件是主观的、情境的——AI 不知道"我是新手"在多大程度上影响决策 | AI 的推荐可能技术上正确但不适合我 |
| **Step 6: 质量检查第4项（AI编造核验）** | AI 检查 AI 有同类盲区——两个 AI 可能犯同样的错误 | 可能放过编造的内容 |
| **信息盲区标注** | "我不知道什么"是一种元认知判断，AI 不知道我的知识边界 | AI 可能列出技术性的已知未知，但遗漏我个人的未知未知 |
| **最终推荐决策** | 决策的责任在我——AI 可以给建议，但"选哪个"必须是我自己的判断 | AI 的推荐基于公开数据，不知道我的隐性偏好 |

**核心原则：AI 负责广度（搜得全、拆得细），人负责深度（判断力、约束匹配、盲区意识）。**

## 6. 未来如果升级，可以增加哪些自动化能力？

| 升级方向 | 描述 | 需要的技术能力 |
|---------|------|---------------|
| **自动化信息检索** | Python 脚本自动调用 Firecrawl API + GitHub API，按预定维度抓取，不再手动执行 | Python + API 调用 |
| **定时监控** | 对已调研的领域设置定期（每周/每月）自动重检索，发现新数据自动对比并标记变化 | Cron/task scheduling + 数据库存储 |
| **多源交叉验证** | 自动对比不同信息源对同一维度的描述是否一致，标记矛盾 | NLP 文本相似度 + 结构化对比 |
| **交互式决策辅助** | 将调研流程做成 Gradio/Streamlit 交互工具，用户填约束 → 自动跑流程 → 输出报告 | Web UI 框架 + 工作流编排 |
| **决策历史追踪** | 记录每次调研的"选择 + 理由"，3个月后回溯——当时的判断是否正确？形成判断力反馈闭环 | 数据库 + 数据分析 |
