# 输出模板：技术调研报告标准结构

> 戴昊杰 (DEEP007) | 2026.07.05 | Task 003
>
> **使用说明：** 本模板是"AI 信息检索与深度调研工作流"的标准输出格式。每次完成调研后，按此结构填充内容。方括号 `[...]` 为占位符，实际使用时替换。

---

## 报告结构概览

```
# [调研主题标题]
  → 第1章：调研动机与范围
  → 第2章：生态概览
  → 第3-N章：候选逐项深度分析（每个候选一章）
  → 第N+1章：横向对比矩阵
  → 第N+2章：个人判断与推荐
  → 第N+3章：学习/行动路线图
  → 第N+4章：信息盲区与限制
  → 来源列表
```

---

## 第 1 章：调研动机与范围

```markdown
## 调研动机

我为什么要做这个调研？（1-2段）

- 我面临的真实问题是什么？
- 不做调研直接行动的风险是什么？

## 调研范围

| 维度 | 说明 |
|------|------|
| 调研主题 | [一句话描述] |
| 候选范围 | [列出所有候选，每个候选一句话简介] |
| 数据截止日期 | [YYYY.MM.DD] |
| 调研约束 | [个人技术水平、预算、偏好、限制] |
| 调研目标 | [最终要做的决定是什么] |

## 调研方法

本次调研使用"AI 信息检索与深度调研工作流 v1.0"：
1. Firecrawl 多源搜索 + GitHub API 抓取
2. Claude Code 结构化分析与提取
3. 人工判断注入（约束条件 + 盲区标注）
4. 横向对比矩阵生成
5. 质量检查（10 项检查表）
6. 结构化报告输出
```

## 模板使用说明

1. **每次使用时复制本模板**，将 `[...]` 占位符替换为实际内容
2. **"我的判断"段落必须亲自写**，不能由 AI 代写——这是整个报告中唯一标注个人声音的地方，也是区分"AI调研报告"和"我做的调研报告"的关键
3. **来源列表不要省略**——如果某个来源后来失效了，至少知道是哪个 URL
4. **盲区标注不能空**——如果一份报告没有任何盲区，恰恰说明它有问题（要么太浅、要么AI编造了确定性）
