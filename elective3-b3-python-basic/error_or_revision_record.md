## 报错记录：Unicode 编码错误

**报错内容：**
```
UnicodeEncodeError: 'gbk' codec can't encode character '✅' in position 0:
illegal multibyte sequence
```

**我一开始以为是什么原因：** 以为是 Python 文件本身有语法错误——检查了代码逻辑没发现问题。

**我如何向 Claude Code 提问：**
```
运行 Python 文件时出现这个错误，代码本身看起来没问题：
UnicodeEncodeError: 'gbk' codec can't encode character '✅'
这是什么原因？怎么修复？
```

**Claude Code 给出的解释：**
- GBK 是 Windows 中文版的默认终端编码
- GBK 不支持 emoji 字符（如 "✅"）
- Python 的 print() 输出到终端时，终端无法渲染 emoji
- 三种修复方案：1) 替换为 ASCII 文本 2) 添加 `# -*- coding: utf-8 -*-` 声明 3) 设置终端编码

**我最终如何修改：**
选择方案 1——把所有 emoji 替换为纯文本：
- "✅" → "[DONE]"
- "⬜" → "[ ]" 或 "[TODO]"

**为什么选择方案 1 而不是方案 2/3：**
- 方案 2（UTF-8 声明）可能在某些 Windows 环境仍然失败
- 方案 3（改终端编码）需要每次手动设置，不可靠
- 方案 1 最简单可靠——纯 ASCII 文本在任何终端都能显示

**我从这个报错中学到了：**
1. Python 代码"能跑"和"能在任何环境跑"是两回事——编码问题是跨平台开发最常见的坑
2. Windows 和 Linux/Mac 的终端编码差异——以后写代码尽量避免 emoji
3. **"选择最简单可靠的方案"**——不要为了保留 emoji 搞复杂的编码设置，纯文本就够用了

## 修改记录："我的理解"注释个人化

**原始代码：** Claude Code 在每个练习文件末尾写了"我的理解"注释，但措辞比较通用——像教科书解释。

**我修改了哪里：** 用自己的话重写了所有"我的理解"注释，加入了：
- 我个人容易理解记忆的类比（"变量 = 一个有名字的盒子"）
- 我的实际使用场景（"列表 = 购物清单"）
- 我真正关心的价值（"循环的价值 = 不用复制粘贴 10 行"）

**为什么这样修改：** Claude Code 的"理解"不是我的理解——它的措辞像老师在讲课。我用自己的话重写后，这些注释才真正反映了我此刻对 Python 的理解水平。**这也是"AI 辅助但不替代思考"的体现。**

**从这个修改中学到的：**
- AI 生成的注释可以当起点，但不能当终点——需要用自己的话验证自己是否真的懂了
- 如果我能用自己的话解释一个概念，才算真的理解——这就是费曼学习法
