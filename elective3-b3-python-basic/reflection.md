## 1. 我在任务前对 Python 的理解是什么？

"一种编程语言，做 AI 和数据分析的人都在用，但跟我没关系——我又不是程序员。"

坦白说，我之前一直觉得 Python 是"别人的工具"——程序员用、数据科学家用、搞 AI 研究的人用。我一个做 AI 应用的学员，用 Claude Code 聊天就够了，不需要学编程。

## 2. 完成任务后，我对 Python 的理解发生了什么变化？

**Python 不是"编程语言"，是"AI 的执行层"。**

做完 B3 后我清楚看到了一个分层：
- **思考层：** Claude/ChatGPT（AI 帮你想、分析、决策）
- **执行层：** Python（把你和 AI 的共识变成可运行的结果）
- **展示层：** HTML/GitHub（把结果变成可展示的资产）

这三层中，Python 是"把想法变成行动"的那一层。没有它，AI 永远只停留在聊天窗口里。

**核心认知变化：我不需要成为 Python 程序员，但我需要能用 Python 让 AI 的工具能力发挥出来。** 就像我不需要是电工，但需要知道"插头插插座"才能用电器。

## 3. Claude Code 在学 Python 中具体帮了我什么？

1. **生成代码（跨越"我不会写"的障碍）：** 我描述"做一个 DEEP 营任务清单生成器"→ Claude Code 生成了完整的 Python 文件。没有 Claude Code，我根本不知道从哪里开始写。
2. **解释概念（降低理解难度）：** print/变量/列表/循环/函数——这些概念对应我的真实需求后，比看书上的抽象定义容易懂 10 倍。
3. **修复报错（加速学习循环）：** Unicode 编码错误——放在以前，我可能搜 30 分钟才能找到原因。Claude Code 10 秒给解释+3 种修复方案。

**最重要的帮助：降低了"试一试"的心理门槛。** 我知道即使出错，Claude Code 能帮我修复——所以更愿意动手跑代码。

## 4. 我是否只是复制代码，还是理解了代码含义？

**80% 理解，20% 会用但不完全懂。**

- `print()` ✅ 完全理解
- `变量 = 值` ✅ 完全理解
- `列表 = [...]` ✅ 完全理解
- `for item in list:` ✅ 完全理解
- `def function():` ✅ 基本理解——知道是"定义一个可复用的代码块"
- `enumerate(tasks, 1)` ⚠️ 会用，但不完全理解参数含义
- `f"{variable}"` ⚠️ 会用，但不确定和 `"..." + variable` 的区别

**"能用但不完全懂"的部分是我下一步要补的。** 第一期目标不是 100% 理解——是能跑通、能改参数、能描述需求让 Claude Code 帮写。
